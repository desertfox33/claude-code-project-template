# /review

Review the requested files, directory, or branch.

Arguments: `$ARGUMENTS`

## Instructions

Use the code-review skill.

Focus on:
- Correctness
- Security
- Reliability
- Maintainability
- Test coverage
- Operational risk

Return:
1. Executive summary
2. Findings by severity
3. Suggested fixes
4. Tests or checks to run
5. Residual risk
