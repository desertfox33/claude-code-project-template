# Code Review Workflow

Use this workflow for branch, PR, or diff review.

## Steps

1. Read the diff.
2. Identify the intent of the change.
3. Use the code-review skill.
4. Use the security-review skill for sensitive changes.
5. Check test coverage.
6. Report findings by severity.

## Output

- Summary
- Findings
- Recommended fixes
- Tests or checks
- Residual risk
