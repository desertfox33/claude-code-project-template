# Feature Build Workflow

Use this workflow for a new feature or meaningful enhancement.

## Steps

1. Understand the request and constraints.
2. Inspect relevant files.
3. Propose a short implementation plan.
4. Make small, focused changes.
5. Add or update tests.
6. Run relevant checks.
7. Summarize the diff, testing, risks, and follow-up work.

## Quality gates

- No secrets committed.
- Security-sensitive behavior reviewed.
- Tests added for changed behavior where practical.
- Rollback or operational risk documented when relevant.
